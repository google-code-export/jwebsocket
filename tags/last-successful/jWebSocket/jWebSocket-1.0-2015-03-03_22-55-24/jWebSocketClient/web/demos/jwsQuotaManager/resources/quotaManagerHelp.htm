Overview

The QuotaManager application is a web solution that enables efficient management of quotas in JWebSocket server. It is a powerful tool for administrators and users, managed easily, efficient and friendly quota, either to create, delete, modify or simply view.

General Structure

The application initially has a form that requests authentication for logging in, users can access only having access permission. After a user logs into the system tab structure appears with the different functionalities very easy to use, these tabs are:

1.    Create Quota: Allows the creation of quotas.

2.    Quota Admin: Allows searching by different filters, modification and removal of quotas and register new quotas to existing ones.   

3.    Help: Helps how use a QuotaManager application.

It also has a button to “Disconnect” in the upper right corner to close the connection. To the left of the bottom bar is the area to display messages generated by the application and in right the user information that logged in that time and “logout” button to user logout.

Tab “Create Quota”

This tab has the functionality to create a quota for a user or group of users defined, to execute certain functionality in jWebSocket server.

Figure 1: Create Quota

The text fields to be filled to create the quota are:

·        Select Quota: When this selection field is deployed, shows the types of quota that the administrator defined in configuration files can be created. Each quota type has a different behavior, among them are the type of quotas:

o   CountDown: As the name suggests their behavior is decreasing, each time the quota is executed its value is decremented.

o   Interval:  Its value is decremented when the quota is executed in an interval of time, and has the characteristic that its value is reset to initial value when starts a new interval cycle.

o   DiskSpace: Quota to restrict the disk space.

·        Value: Numeric value that gives to a quota, indicating the number of times that the quota can be executed. 

·           Instance: The value of this attribute represented in who is instantiated the quota, at first sight it may seem that this attribute should be called user, but quotas are designed to be completely flexible and can be applied to both, users and user groups, but not necessarily restricted to only these two, in future can also be established quotas to functionality and applications.

 

·           InstanceType: Define the instance type of the instance element, so far this element can take two values, "User" when instance is a user and "Group" when instance is a user group.

·           Name Space: Plugin namespace to which the quota will be applied. If the quota type is DiskSpace, this text field is changed by the selection field "Select the folder type".  

 

·           Select the folder type: For quota type DiskSpace, only will be able to choose "private" to execute quota in private folders and "public" to execute quota in public folders.

 

·           Actions: When you want to restrict all actions in the plugin indicated per Name Space, then this value is set to "*". If you only want to restrict some of the features and not all, you must specify separated by a "," the actions that will be restricted by the quota. For example you can specify the following value: "create, delete, update", in this case only these three types of actions to be implemented will be penalized by the quota.

 

After entering all fields in the form, the "Create Quota" button is pressed to create the quota, if the data are satisfactory and the data is saved, in the message area a confirmation message appears, if they are not satisfactory in the message box displays an error message. Also the form has a "Reset" button to clear the form data.

 

Tab “Quota Admin”

 

·         Search Quotas

Search quotas it can do by three criteria fields: Quota Type, Name Space and Instance. This search can be by one of these criteria, two or simply 3 together. 

                              

Figure 2: Search criteria

To search you must enter at least one criteria and following pressing the "Filter" button, which will display the data in the table of quotas which meet those criteria, if any.                   

 

Figure 3: Example search

The quotas on the table may contain registered quotas which can be displayed by clicking on the symbol "+".   These quotas may be modified or eliminated through the icons to the right of each.

Figure 4: Example quota registered

 

 

·         Modify a quota

 

To modify the value of a quota must perform a search, which display a series of quotas. Of the quotas show you, select that you want change its value and click on the "Alter Quota" button.   When you do this action a form is displayed, to the right side of form, the Quota Information area, with the information of selected quota and in the left side an area to perform different actions with the quota value.     

 

Figure 5: Modify quota

Actions to modify the value of quotas:

o   Get Quota Value: pressing “Execute” button the value of the quota is shown. This text field is not editable because it shows only the current value of the quota

o   Set Quota Value: in this text field must be set the new value that will have the selected quota, after that click on Execute”button, if is correct in the text field Get Quota Values shown the new quota value.

o   Increase Quota Value in this text field must be set the value that will increase the present value of the selected quota, after that click on “Execute” button, if is correct, in the text field Get Quota Values shown the new quota value.

o   Reduce Quota Value: in this text field must be set the value that will decrease the present value of the selected quota, after that click on “Execute”button, if is correct, in the text field Get Quota Values shown the new quota value.

 

·           Remove a quota

To remove quota you must perform a search which displays a list of quotas. Of the quotas shown, select that you want to delete and click the "Unregister Quota" button. When you do this action appear a confirmation quota remove message.

 

·           Register a quota

To register quota must perform a search which displays a list of quotas.   When selecting a quota listing, click on "Register Quota" button, a form is displayed, to the right side of this, an area with the quota information selected and in left side an area with a text field Instance. After filling the form click on “Register Quota” button to creating a new quota registered with the selected one.
